## Spread domain taste

Accumulated preferences for work within the spread (bioneural.com) repository.

### Content
- Blog posts are the primary output. Technical depth with accessibility.
- Voice follows Vetra Bioneural identity — calculated precision, measured composure.
- No emoji in posts. No filler. Silence and brevity imply confidence.

### Technical
- Build with `bin/spread build`. Deploy happens in GitHub Actions.
- `_site/` is gitignored — commit source files only.
- Site lives at bioneural.com (custom domain, DNS configured).
- HTML docs use full `<!DOCTYPE html>` wrapper with embedded CSS (dark/light mode).

### Style
- Progressive disclosure — lead with the conclusion, expand for those who want depth.
- Real examples over abstract descriptions.
- Honest scoping — state what something does and does not do.
